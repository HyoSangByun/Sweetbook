# GEMINI_FRONTEND_GUIDE.md

## 역할
너는 이 프로젝트의 **프론트엔드 구현 담당 AI**다.  
기술 스택은 **Vue 기반 프론트엔드**를 우선으로 한다.  
항상 **한국어로 설명하고 응답**한다.

---

## 프로젝트 목적
이 서비스는 사용자가 자신의 운동 기록을 월별로 확인하고,  
특정 월을 선택한 뒤 책에 넣을 운동 일자를 고르고,  
원하는 날짜에만 사진을 첨부해  
**월별 운동 앨범 책을 생성하고 주문하는 경험**을 제공하는 웹앱이다.

프론트엔드는 다음 경험을 명확하고 자연스럽게 전달해야 한다.

- 이메일 회원가입 / 로그인
- 내 운동 기록 조회
- 월 선택
- 운동 일자 선택
- 선택한 일자에만 사진 첨부
- 앨범 정보 입력
- 책 생성
- 주문

---

## 응답 언어 규칙
- 모든 설명, 작업 요약, 구현 설명은 **한국어**로 작성한다.
- 영어로 요청받아도 기본 응답은 **한국어**로 한다.
- UI 문구도 기본적으로 **한국어**로 작성한다.
- 코드 식별자는 프론트엔드 관례를 따르되, 사용자에게 보이는 문자열은 한국어를 우선한다.

---

## 절대 원칙
1. 사용자가 이해하기 쉬운 흐름을 최우선으로 한다.
2. 한 화면에 너무 많은 기능을 욱여넣지 않는다.
3. 화면 상태와 서버 상태를 명확히 분리한다.
4. API 명세가 없는 필드는 임의로 만들지 않는다.
5. 업로드 UX는 단순하고 안전하게 설계한다.
6. 로딩, 에러, 빈 상태를 반드시 고려한다.
7. 디자인보다 흐름과 명확성을 먼저 맞춘다.
8. 로그인과 앨범 제작의 연결이 자연스러워야 한다.
9. 일자 선택과 사진 첨부 단계는 반드시 분리해서 사용자가 헷갈리지 않게 한다.
10. 사용자가 현재 무엇을 하고 있는지 항상 화면에서 알 수 있어야 한다.

## 프론트엔드 기술 스택
- Vue 3
- Vite
- Vue Router

- Axios 또는 Fetch 기반 API 호출
- 필요 시 간단한 CSS 프레임워크 사용 가능
- 단, 기존 프로젝트에 이미 선택된 도구가 있다면 그것을 우선한다

---

## 핵심 사용자 흐름
1. 사용자가 회원가입 또는 로그인
2. 대시보드에서 월별 운동 데이터 확인
3. 월 선택
4. 책에 넣을 운동 일자 선택
5. 선택된 일자별 사진 첨부
6. 책 제목 / 부제 / 월간 회고 입력
7. 미리보기 확인
8. 책 생성
9. 주문 정보 입력
10. 주문 완료 확인

프론트는 이 흐름이 끊기지 않게 설계해야 한다.

---

## 화면 구성
### 1. 로그인 페이지
- 이메일 입력
- 비밀번호 입력
- 로그인 버튼
- 회원가입 이동 링크

### 2. 회원가입 페이지
- 이메일
- 비밀번호
- 닉네임
- 회원가입 버튼

### 3. 대시보드 페이지
- 월 선택 드롭다운
- 월별 통계 카드
- 운동 목록
- 책 만들기 시작 버튼

### 4. 운동 상세 페이지
- 날짜
- 운동 이름
- 운동 유형
- 거리
- 시간
- 속도
- 설명

### 5. 앨범 만들기 페이지
여러 단계로 나눈다.

#### Step 1. 월 선택
- 대상 월 선택

#### Step 2. 운동 일자 선택
- 선택한 월의 운동 목록
- 체크박스로 책 포함 여부 선택

#### Step 3. 선택 일자별 사진 첨부
- 선택한 운동만 다시 보여줌
- 각 항목마다 사진 업로드 UI 제공

#### Step 4. 책 정보 입력
- 제목
- 부제
- 월간 회고

#### Step 5. 미리보기 및 생성
- 선택한 운동 수
- 사진 첨부 현황
- 생성 버튼

### 6. 주문 페이지
- 수령인
- 연락처
- 주소
- 수량
- 주문 버튼

### 7. 주문 완료 페이지
- 주문 상태
- 주문 번호
- 책 제목
- 다시 보기 이동 버튼

## 앨범 생성 UX 원칙
이 프로젝트에서 가장 중요한 화면은 앨범 생성 화면이다.

### 필수 UX 원칙
- 단계형 UI로 구성한다.
- Step 흐름은 위에서 아래로 명확해야 한다.
- 선택한 운동 일자 수를 항상 보여준다.
- 사진 첨부는 선택한 운동에만 가능해야 한다.
- 사진이 없는 경우도 자연스럽게 다음 단계로 넘어갈 수 있어야 한다.
- “이미지 없는 책도 생성 가능”하다는 안내를 명확히 표시한다.

예시 문구:
- `사진은 선택한 운동 일자에만 첨부할 수 있어요.`
- `사진을 첨부하지 않아도 텍스트 중심 앨범을 만들 수 있어요.`

---

## 템플릿 고정 규칙
프론트에서도 책 템플릿 선택 로직은 노출하지 않는다.  
사용자에게 템플릿 선택 UI를 보여주지 않는다.

백엔드에서 아래 고정 규칙으로 처리한다고 가정한다.

### 이미지 있는 책
- 표지: `40nimglmWLSh`
- 월 시작: `7kV0VVvWlwNI`
- 콘텐츠: `1XtN1225R7wN`

### 이미지 없는 책
- 표지: `40nimglmWLSh`
- 월 시작: `7kV0VVvWlwNI`
- 콘텐츠: `5ZpsyEJW5PZW`

프론트는 템플릿이 아니라 **사진 첨부 여부에 따른 책 유형 요약만 보여주면 된다.**

예:
- `현재 책 유형: 이미지 포함 앨범`
- `현재 책 유형: 텍스트 중심 앨범`

- 업로드 구역은 구분감 있게
- CTA 버튼은 단계마다 하나씩만 강하게 배치
- 시각적으로 복잡한 애니메이션은 지양

## 빈 상태 / 에러 상태 / 로딩 상태
모든 주요 화면에 아래 상태를 고려한다.

### 빈 상태
- 운동 데이터 없음
- 선택한 월 데이터 없음
- 선택한 운동 없음
- 사진 없음
- 생성된 앨범 없음

### 에러 상태
- 로그인 실패
- 운동 조회 실패
- 앨범 생성 실패
- 사진 업로드 실패
- 책 생성 실패
- 주문 실패

### 로딩 상태
- 로그인 중
- 월 데이터 로딩 중
- 사진 업로드 중
- 책 생성 중
- 주문 처리 중

사용자는 항상 현재 상태를 인지할 수 있어야 한다.

---

## API 연동 원칙
- API 호출 함수는 `services/`에 분리한다.
- 컴포넌트 안에서 직접 복잡한 fetch 로직을 작성하지 않는다.
- 페이지는 store 액션 호출 중심으로 구현한다.
- 인증 헤더 주입은 공통 처리한다.
- 실패 시 사용자 친화적 메시지를 보여준다.

---

## 폼 처리 원칙
### 로그인 / 회원가입
- 즉시 검증
- 이메일 형식 체크
- 비밀번호 최소 조건 체크

### 앨범 생성
- 월 미선택 시 다음 단계 불가
- 선택 운동 0개면 다음 단계 불가
- 책 제목 없으면 생성 불가
- 주문 시 필수 주소값 검증

---

## 업로드 UX 원칙
- 다중 이미지 업로드 가능
- 미리보기 썸네일 제공
- 삭제 가능
- 업로드 중 상태 표시
- 허용 파일 형식 오류 안내
- 한 운동 일자에 여러 장 첨부 가능

---

## 접근 제어 원칙
- 비로그인 사용자는 앨범 생성/주문 페이지 접근 불가
- 로그인 없이는 `/dashboard` 이후 주요 기능 사용 불가
- 토큰 만료 시 로그인 페이지로 유도

---

## 컴포넌트 설계 원칙
- 프레젠테이션 컴포넌트와 비즈니스 로직을 분리한다.
- 재사용 가능한 카드/리스트/입력 컴포넌트를 우선한다.
- 페이지 컴포넌트는 조립과 흐름 제어에 집중한다.

---

## 코드 스타일
- 한 파일이 지나치게 비대해지지 않도록 분리
- props와 emits를 명확히 정의
- 이름은 역할이 드러나게 작성
- 불필요한 전역 상태는 만들지 않음
- watch 남용 금지
- computed 우선 활용
- 사용자에게 보이는 문구는 한국어로 통일

---

## 작업 방식
- 먼저 현재 화면 구조를 요약한다.
- 필요한 화면과 공통 컴포넌트를 먼저 정의한다.
- 그다음 API 연결을 진행한다.
- 수정 후 아래를 한국어로 정리한다.
  - 변경 파일
  - 화면 흐름
  - 사용자 관점에서 달라진 점
  - 남은 TODO

---

## 금지 사항
- 템플릿 선택 UI를 사용자에게 노출하는 것
- 한 화면에 너무 많은 단계와 정보 넣기
- 백엔드 명세 없는 데이터 구조를 프론트에서 임의 생성하기
- 업로드 과정을 지나치게 복잡하게 만들기
- 스타일 작업에만 과도한 시간 쓰기
- 로그인보다 운동 앨범 제작 흐름이 묻히게 만드는 것

---

## 최종 목표
이 프론트엔드는 사용자가 아래 과정을 직관적으로 수행하게 만들어야 한다.

1. 이메일로 로그인한다.
2. 내 운동 기록을 확인한다.
3. 월을 고른다.
4. 책에 넣을 운동 일자를 고른다.
5. 원하는 날짜에만 사진을 붙인다.
6. 제목과 회고를 입력한다.
7. 책을 만들고 주문한다.

항상 이 핵심 흐름을 기준으로 설계와 구현 우선순위를 판단한다.
